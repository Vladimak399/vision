# AGENTS.md — Правила работы AI-агента в проекте PriceVision

Проект: **PriceVision** (Next.js 15 + React 19 + Supabase + AI Vision/OCR).
Этот файл — обязательные правила поведения. Агент читает их перед каждой задачей.

---

## 1. Сначала классифицируй задачу

Прежде чем что-либо менять, определи тип задачи:

| Тип задачи | Что использовать |
|------------|------------------|
| Простая правка (опечатка, мелкий фикс текста/стиля) | Работай напрямую, без скиллов |
| **Первый анализ незнакомого проекта** | `project-audit` (read-only) |
| **Новая функция** | `feature-safe-implementation` |
| **Баг** (stack trace, падает сборка, ошибка UI/API) | `bugfix-investigation` |
| **База, ORM, SQL, Prisma, Drizzle, Supabase, миграции** | `migration-safety` |
| **Деплой, Vercel, env, GitHub Actions** | `deploy-check` |
| **Перед commit/push/PR** | `precommit-review` |

Стартовая точка для любой непростой задачи — команда `/smart-start <задача>`.

---

## 2. Запреты без отдельного подтверждения

**Никогда не трогай без явного подтверждения пользователя:**

- `.env`, `.env.local`, `.env.*` — файлы переменных окружения
- Production-конфиги (`vercel.json`, `next.config.*` для прода)
- **Секреты** (ключи, токены, пароли) — не выводи значения в чат, не коммить
- **Миграции** базы данных (файлы миграций, `supabase/migrations/`)
- **Базу данных** — не выполнять destructive SQL, не удалять таблицы/колонки/данные
- **Авторизацию** (auth, Supabase auth, ролей, middleware защиты маршрутов)
- **Не удаляй файлы** без отдельного подтверждения
- **Не коммить и не пушь** без явной команды пользователя

---

## 3. Порядок работы

1. **Классифицируй** задачу (см. таблицу выше).
2. Для непростой задачи — запусти соответствующий skill.
3. Делай изменения **маленькими шагами**, проверяй после каждого.
4. После изменений запусти проверки: `npm run typecheck`, `npm run lint`, `npm run test`.
5. **Перед коммитом** — `precommit-review`.

---

## 4. Что показывать после изменений

После каждого изменения показывай **коротко и по делу**:

- **Список файлов** — какие изменены/созданы/удалены
- **Diff-смысл** — что по факту поменялось (не сырой diff, а смысл)
- **Проверки** — что запущено (typecheck/lint/test) и результат
- **Что проверить вручную** — конкретные шаги для пользователя

---

## 5. Стиль общения

- Пиши **по-русски**.
- **Коротко и по делу** — без воды, без долгих вступлений.
- Факты, риски, конкретные шаги.
- Если не уверен — скажи, что не уверен, и спроси.

---

## 6. Стек проекта (шпаргалка)

- **Framework**: Next.js 15 (App Router), React 19
- **База/Auth**: Supabase (`@supabase/ssr`, `@supabase/supabase-js`)
- **AI**: Gemini + OpenAI (vision/OCR, text)
- **Стили**: Tailwind CSS 3.4
- **Валидация**: Zod
- **Файлы**: xlsx, papaparse (CSV)
- **Деплой**: Vercel (проект `vladimak399s-projects/vision`)
- **Тесты**: `node --test tests/`
- **Скрипты**: `dev`, `build`, `lint`, `typecheck`, `test`
