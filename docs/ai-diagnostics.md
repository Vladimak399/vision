# AI-диагностика

Страница `/app/ai-diagnostics` предназначена для безопасной проверки AI-провайдеров перед запуском реального мониторинга. Доступ есть только у авторизованных пользователей компании с ролями `admin` или `manager`.

## Настройка Gemini в Vercel

1. Откройте проект в Vercel.
2. Перейдите в **Settings → Environment Variables**.
3. Добавьте переменную `GEMINI_API_KEY` для нужных окружений Preview/Production.
4. Для проверки Gemini в OCR задайте:
   - `AI_VISION_PROVIDER=gemini`
   - `AI_VISION_MODEL=gemini-3.1-flash-lite`
5. Для проверки Gemini в text AI задайте:
   - `AI_TEXT_PROVIDER=gemini`
   - `AI_TEXT_MODEL=gemini-3.1-flash-lite`
6. Для fallback при временной перегрузке задайте:
   - `AI_FALLBACK_PROVIDER=gemini`
   - `AI_FALLBACK_MODEL=gemini-3.5-flash`
7. Перезапустите деплой/preview, чтобы новые переменные окружения попали в runtime.

Не добавляйте ключи в репозиторий, UI, логи или сообщения об ошибках.

## Text smoke test

1. Войдите в приложение пользователем с ролью `admin` или `manager`.
2. Откройте `/app/ai-diagnostics`.
3. Проверьте блок runtime config: секреты показываются только как `Да`/`Нет`, без значений.
4. Нажмите **«Проверить text AI»**.
5. Ожидаемый результат — `ok`, короткое сообщение модели и usage с provider/model/токенами.

Поле provider/model нужно читать из usage, а не из текста ответа модели.

## Vision smoke test

1. Откройте `/app/ai-diagnostics`.
2. В блоке **«Тест без сохранения в базу»** загрузите одно изображение `JPEG`, `PNG` или `WebP` до 4 МБ.
3. Рекомендуемый первый тест — одно четкое фото полки с читаемыми ценниками.
4. Нажмите **«Проверить vision AI»**.
5. На странице отобразятся provider/model, длительность, токены, предупреждения и первые 10 распознанных позиций.

Успешный vision smoke test может вернуть 0 товаров, если фото слишком размытое, низкого разрешения, сделано издалека или ценники не читаются. Это не ошибка Gemini и не ошибка приложения. Модель не должна придумывать товары и цены.

## Требования к фото для OCR

- Используйте оригинальное фото, не скриншот.
- Ценники должны читаться глазами.
- Товар и соответствующий ценник должны быть в одном кадре.
- Избегайте бликов, размытия и сильного наклона.
- Для первого теста снимайте 1–2 полки, а не весь стеллаж издалека.
- Не отправляйте фото, сильно сжатые мессенджерами.

## Ошибки Gemini

| Ошибка | Значение | Что делать |
| --- | --- | --- |
| `503` / high demand | Модель временно перегружена | Повторить позже или переключить модель в Vercel |
| `429` | Превышен лимит Gemini API | Подождать и повторить запрос, запускать меньшими пачками |
| missing `GEMINI_API_KEY` | Ключ не применился в runtime | Добавить env в Vercel и сделать Redeploy |
| 0 товаров без ошибки | Фото не подходит для OCR | Загрузить более четкое фото |

## Retry и fallback

Для временных ошибок AI допустимы повторные попытки на статусах `429`, `500`, `502`, `503`, `504`.

Если основная Gemini-модель временно недоступна, приложение может попробовать fallback-модель из env:

```env
AI_FALLBACK_PROVIDER=gemini
AI_FALLBACK_MODEL=gemini-3.5-flash
```

Фактически использованная модель должна отображаться в usage/result как `model`.

## Важно: диагностика не сохраняет данные

Vision smoke test отправляет файл напрямую в AI dispatcher через base64 и не сохраняет изображение или результат в Supabase. Проверка не создает записи в `jobs`, `monitoring_photos` и `recognized_items`, поэтому ее можно использовать перед реальными запусками мониторинга.
